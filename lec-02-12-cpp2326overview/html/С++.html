c++23: deducing this

explicit object parameter & explicit object member functions

For a non-static non-virtual member function not declared with cv-qualifier or ref-qualifier
struct T {
    void f(this T& self, int i) { ... }
};

deducing this

struct T {
    template <class Self>
    void f(this Self&& self, int i) { }
};

consequences

  • function deduplication

  • recursive lambda

  • simplify CRTP

recursive lambda

int main() {
   auto factorial = [](this auto& self, int i) {
     if (i == 0) return 1;
     return i * self(i - 1);
   };

   std::cout << factorial(0) << ", " << factorial(5);
}

CRTP

template <class Derived>
struct A {

    void run() {
        static_cast<Derived*>(this)->f();
    }

    void f() { std::cout << "A" << std::endl; }
};

struct B : A<B> {
    void f() { std::cout << "B" << std::endl; }
};

struct C : A<C> {};

int main() {
    B b; b.run();
    C c; c.run();
}

CRTP

struct A {
    template <class Self>
    void run(this Self&& self) {
        self.f();
    }

    void f() { std::cout << "A" << std::endl; }
};

struct B : A {
    void f() { std::cout << "B" << std::endl; }
};

struct C : A {};

int main() {
    B b; b.run();
    C c; c.run();
}

function deduplication

#include <iostream>

struct A {
    template <class Self>
    void run(this Self&& self) {
        std::forward<Self>(self).runImpl(); // perfect forwarding
    }

    void runImpl() & { std::cout << __PRETTY_FUNCTION__ << std::endl; }
    void runImpl() && { std::cout << __PRETTY_FUNCTION__ << std::endl; }
    void runImpl() const { std::cout << __PRETTY_FUNCTION__ << std::endl; }
};

int main() {
    A a;
    a.run();            // void A::runImpl() &
    std::move(a).run(); // void A::runImpl() &&

    const A& ca = a;
    ca.run();           // void A::runImpl() const
}

c++23: std::expected

motivation

New error mechanism

  • error visibility

  • clean code

    • monadic interface

    • bridge between throw and nothrow worlds

  • use the return channel without monopolizing it

  • good for compiler optimization

  • handle multiple errors: collect, group or combine

conceptual model

expected<T, E> models a discriminated union of types T and unexpected<E>.
expected<T, E> is viewed as a value of type T or value of type unexpected<E>, allocated in the same storage,
along with observers to determine which of the two it is.

See details here

example

#include <expected>
#include <iostream>

enum class error
{
    compile_time_error,
    runtime_error
};

[[nodiscard]] auto unexpected_runtime_error() -> std::expected<int, error>
{
    return std::unexpected(error::runtime_error);
}

int main()
{
    std::expected<double, int> ex = std::unexpected(3);

    if (!ex)
        std::cout << "ex contains an error value\n";

    if (ex == std::unexpected(3))
        std::cout << "The error value is equal to 3\n";

    const auto e = unexpected_runtime_error();

    e.and_then([](const auto& e) -> std::expected<int, error>
    {
        std::cout << "and_then: " << int(e); // not printed
        return {};
    })
    .or_else([](const auto& e) -> std::expected<int, error>
    {
        std::cout << "or_else: " << int(e); // prints this line
        return {};
    });
}

c++26: reflection

static reflection

  • reflection values (using opaque type std::meta::info)

  • reflection prefix operator ^

  • consteval metafunctions

  • splicers: [: refl :]

example 1

int main() {
    constexpr auto r = ^int;
    typename[:r:] x = 42;       // Same as: int x = 42;
    typename[:^char:] c = '*';  // Same as: char c = '*';

    static_assert(std::same_as<decltype(x), int>);
    static_assert(std::same_as<decltype(c), char>);
    assert(x == 42);
    assert(c == '*');
}

struct S {

consteval auto member_number(int n) {

template<typename E>
  requires std::is_enum_v<E>
constexpr std::string enum_to_string(E value) {
  std::string result = "<unnamed>";
  [:expand(std::meta::enumerators_of(^E)):] >> [&]<auto e>{
    if (value == [:e:]) {
      result = std::meta::identifier_of(e);
    }
  };
  return result;
}

c++26: contracts

contract assertions

Contract assertions allow the programmer to specify properties of the state of the program that are expected to hold at certain points during execution.

assertions example

int f(const int x)
pre (x != 1) // a precondition assertion
post(r : r == x && r != 2) // a postcondition assertion; r names the result object of f
{
contract_assert (x != 3); // an assertion statement
return x;
}

evaluation semantics

  • ignore: do nothing, well-formed predicate

  • observe: call contract-violation handler, continue

  • enforce: contract-violation handler, terminate

  • quick-enforce: terminate
